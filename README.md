# Auto Backup Script

## Включение Скрипта

1. Заполняем полный путь до папки/файла которую нужно архивировать в файл `files.txt`.
2. Выполняем код представленный ниже

```bash
chmod +x ./backup.sh #Задаем ему права на выполение
./backup.sh
```

## Основные блоки кода

Скрипт делится на 3 основных блока:
1. Логирование - за это отвечают функции **err_report()** и **log()**.
2. Архивирование нужной папки/файла - за это отвечает **make_backup()**.
3. Автоматическое удаление лишних бэкапов - за это отвечает **delete_backup()**.

## Логика работы блоков

### 1. Логирование

#### 1.1 **err_report**

В конце программы представлена команда `trap 'err_report $LINENO "$BASH_COMMAND"' ERR`. При возникновении проблемы с какой из либо команд она передаст номер строки (`$LINENO`), в которой возникла проблема, и команду (`$BASH_COMMAND`), в которой возникла проблема, в функцию `err_report()`. Функция в свою же очередь передаст сообщение в лог, с пометкой `[ ERROR ]`.

#### 1.2 **log**

Каждая важная команда вместо того, чтобы передавать свой вывод в `stdout`/`stderror`, передает его в функцию `log()`. Функция в свою очередь, сначала узнает сколько ей передали параметров, если ни одного, то она начнет разделять строки, которые она получила от команды на отдельные и каждую выводить в файл лога с временной меткой (`[ HH:MM:SS ]`), в случае же если она получила 1 и более аргументов то передаем уже их в лог, с той же временной меткой.

### 2. Архивирование

Архивация состоит из двух циклов и сжимания нужной папки.
1. Первый цикл (**while**) - его задача заключается в том, что в него передается `files.txt`, а цикл читает строки, сразу разбивая их в массив **words**, состоящего из имен директории которые имеются на пути к нашему требуемому файлу/папке.
2. Второй цикл (**for**) - его задача заключается в том чтобы постепенно собирать нужную нам строку обратно, и при обнаружении конечного элемента архивировать его в папку `backups`, а не около изначальной папки/файла, который мы архивируем. Сбор строки требовался нам, для того, чтобы указать на папку/файл, которую мы собираемся архивировать.
3. Сжимание - при первой архивации папки/файла мы создаем для него собственную директорию, в которой будут хранится все архивные версии данного файла. Сжимается файл с помощью `.tar.gz`, файл создается с его изначальным названием + метка даты + метка времени.

### 3. Автоматическое удаление

Сначала мы построчно передаем все файлы папки внутри `backups`, в цикл (**while**), который будет построчно передавать все папки в проверку на то, больше ли файлов в папке чем нужно, и если же да, то вызываем функцию `delete_backup()`, передавая в качестве параметра папку из которой нужно удалить лишний архив. Внутри функции используем еще один цикл, в который передаем построчно список файлов, рекурсивно отсортированный, данный список преобразуется в массив `files`, из которого, удалим первый элемент, который будет являться самым старым.
